## Reaching Universalis

The goal of reaching universalis is to have a Open world sandbox / Life sim

I want to be able to set up a game in multiple different worlds. IE Naurto, Battletech, WW2, modern day, Gate: JSDF. and then enter the world and play within the world

I want it to be NPC focused as in the world should feel alive and like the player isnt important. i want the player to have access to the same tools that the NPCs have and the NPCs to have access to the same tools the player has.

## Clustering/Multiplayer

I want the game/sim to be clusterable and multithreaded, so that you can off load the agent treads to the compute blades allowing high detailed agents at a decent speed. i also want it to be p2p multiplayer.

When a update happens i want the update to be downloaded via the p2p network

## UI

I want the graphics to be simple, this will free resources to be put into the sim, and also allows the sim to be repurposed to allow completely different genres to be playable. 

the UI should be Data focused and have all the data be availiable as long as the player would know it

the UI the player interacts with should be unattached to the simulation, so that if the sim is lagging behind the player still has a responcive UI/UX to interact with

The main UI screen should be a Map view like Google maps. the play should be able to scroll around and buttons should open up new windows that will allow the player to buy/sell, issue orders, look at finances, etc.
